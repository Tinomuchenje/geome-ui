About the Genomic Observatories Metadatabase

The Genomic Observatories Meta-Database (GeOMe) is a web-based database which captures metadata on biological samples, used for biodiversity inventories, population studies, and environmental metagenomics. GeOMe satisfies the requirements of the genomic observatories model, including capturing the who, what, where, and when associated with all samples. GeOMe draws on both the Darwin Core and the Minimum Information about any type of Sequence (MIxS) standards.


Welcome to GeOMe! There are several routes you can take to get started with GeOMe. The route you take depends on whether you are interested in only adopting an available template, generated from a network approved configuration, or if you want to submit data to the GeOMe repository. A network approved configuration is a configuration template that projects can adopt during creation. Currently, there are two network approved configurations: one for Biocode projects and another for GeOMe publications (which follow the model developed by the Diversity of the IndoPacific Network for publishing genomic data. Templates are a collection of data attributes, rules, and lists that are represented through the interface as JSON files, with the option of generating Excel spreadsheets. JSON files are useful for developers in automating the discovery of metadata attributes belonging to a template while Excel spreadsheets are useful for guiding the data attributes for a particular task.

If you choose to submit data to the GeOMe repository, you can choose to submit to an existing project, create a new project that is synchronized to a network approved configuration, or create a new project either from scratch or cloned but un-synchronized with a network approved configuration. When a project is synchronized with a network approved configuration, all changes to the master configuration (e.g. Biocode), will apply to all projects that belong to the master configuration. Projcts that are cloned but un-synchronized with the network approved configuration will not receive any configuration updates once they are created.

The following lists the available routes you can choose to get started:

Route 1: I want to query public data or generate a template
Most GeOMe projects are marked as "public", which means that anyone can query public data or generate that project's templates. Read the GeOMe Help Document to get started using the web interface for generating templates and querying data.

Route 2: I want to contribute data to an existing project
See the GeOMe projects page to read about current GeOMe projects. Contact the project administrator of the project you would like to join and the will either create a new account for you or associate your existing account to their project. Read the GeOMe Help Document to get started using the web interface.

Route 3: I want to create a new project, synchronized with a network approved configuration
If you choose to create a project that is synchronized with a network approved configuration, you will receive all updates and additions of data attributes, rules, and lookkup lists of the network approved configuration. Creating any new project currently requires sending an email request to geome.help@gmail.com with a brief statement of what your project involves.

Route 4: I want to create a new project, un-synchronized with a network approved configuration
If you choose to create a project that is un-synchronized with a network approved configuration, you will either start from scratch or you can choose to clone a network approved configuration. If your project is un-syncrhonized, then you are free to make your own changes and additions to your project configuration and you will not receive any changes made by the master network approved configuration. Creating a new project currently requires sending an email request to geome.help@gmail.com with a brief statement of what your project involves.

GeOMe's web interface lets users create spreadsheet templates, customized for individual projects, which ensure rapid feedback on data quality during the data loading process. The web interface lets users upload data on collecting events, specimens, tissues, photos of specimens and events, FASTA files, and FASTQ metadata. Users can also choose to create their own GeOMe projects and customize their own attributes, lookup lists, rules, and project members.

Step by step instructions on using the GeOMe web interface are available by reading our GeOMe Help Document (PDF format).

Data uploaded to GeOMe can be manipulated in a laboratory information management system (LIMS) using a specially built LIMS plugin that operates within the Geneious environment. The purpose of the LIMS tool is to help manage lab and sequence analysis workflows.Use the LIMS plugin wiki to learn about how to use the LIMS and download the latest version from the Biocode LIMS Release page. After you have installed the plugin and learned how to open a connection from the wiki docs, specify the following to make your connection:

  • Field Database Connection: GeOMe FIMS
  • Host: https://api.develop.geome-db.org (currently only the development version of the LIMS is specified).
  • Username/password: You will need to obtain a username and password in order to make any connection to the LIMS system.

GeOMe has its roots in the Moorea Biocode project database, developed from 2006 to 2011 to support data collection for the Moore Foundation funded Moorea Biocode Project: an all taxa biotic inventory of a single tropical island involving 6 teams, 50 researchers, and thousands of collecting events. The Moorea Biocode field information management system, also known as "FIMS1", was developed to ingest spreadsheet data from researchers working on the project and employed data validation on data ingest. The tools were written on server side with Perl with a client application in Java (bioValidator). FIMS1 has been running from 2006 through 2018 and developed tools such as the Plate Matcher (to easily map tissues to 96 well plates), the bioValidator (for loading and validating spreadsheets), and a web interface for managing data.

From 2012 to 2015, the National Science Foundation funded the BiSciCol (Biological Sciences Collections) project. While BiSciCol focused on identifiers, ontologies, and semantic technologies for linking biodiversity data, the primary use case was to work on a solution for linking events, samples, and tissues across many different systems using linked data technology. The project had mixed results but did result in several achievements: a clearer understanding of the role of persistent identifiers, building a darwin core triplifier, the development of the Biological Collections Ontology (BCO), and FIMS2 (running at http://biscicol.org/). FIMS2 collaborated with the California Digital Library system to assign ARK identifiers (through their EZID system) for all samples, events and tissues. In addition, all data inserted into FIMS2 was backed by a Fuseki triplestore with all metadata directly assigned data attributes identified by Darwin Core and the MIxS Standards.

From 2014-2017, FIMS2 also began working with the Diversity of the IndoPacific Network (DIPNet) to develop a coherent metadata repository for assembling metadata from across the IndoPacific region. DIPNet added the ability to upload FASTA and FASTQ metadata to the FIMS2 site and also developed the name "GeOMe", which eventually became the brand name for FIMS3. As of 2018, FIMS2 is still running at http://biscicol.org/ but is being transitioned to FIMS3.

Beginning in 2016, John Deck and RJ Ewing started work on a redesigned FIMS system, utilizing a Postgres backend to control project management features and used JSON metadata objects to store data and configurations. This FIMS system (FIMS3) assumed the name GeOMe. It incorporates design features from FIMS1 (including photo uploading, specimen, tissue, and event pages, plate-matching tools) along with features of FIMS2 (persistent id generation, flexible project configuration with attributes stored in a non-relational system guided by a configuration file). In addition, FIMS3 brings in the concept of a network which governs multiple configuraiton templates and the ability to easily create projects. GeOMe will be replacing FIMS1 and FIMS2 instances before the end of 2018.

The Genomic Observatories Metadatabase has the following objectives:
  • To advance genetic diversity research worldwide
  • To aggregate sample and genetic data in raw formats in a searchable database such that original datasets can be utilized for further investigation.
  • To promote and advocate open and collaborative science as a best practice for conducting biodiversity research
  • To promote and provide capacity-building within developing countries for monitoring, study and protection of their biodiversity resources.

Recalling that access to and utilization of genetic resources and data taken should be consistent with the provisions of the Convention on Biological Diversity (CBD) taking into account their specifications by the Bonn Guidelines on Access to Genetic Resources and the Fair and Equitable Sharing of Benefits arising from their Utilization, and, where appropriate, the Nagoya Protocol on Access to Genetic Resources and the Fair and Equitable Sharing of Benefits arising from their Utilization (NP),

Recalling that according to these provisions non-monetary and/or monetary benefits from the utilization of the genetic resources shall be shared with the Country of Origin if the same so requires and as it is set out in mutually agreed terms,

Acknowledging that research and development on genetic resources can be for the public domain (non-commercial) or for commercial purposes and,

Recalling that according to these provisions, non-commercial research purposes may contribute to the conservation and sustainable use of biodiversity

By uploading data to or downloading data from the DIPnet database, DIPnet contributors and Recipients of DIPnet data agree as follows:

ARTICLE 1 – UPLOADING OF LEGAL AND ACCURATE GENETIC DATA AND ASSOCIATED METADATA

By uploading Genetic Data and associated metadata to GeOME database, Contributors certify the following:

1.1 The Genetic Resources from which Genetic Data and Associated Metadata were derived were collected under appropriate and legal access permits or their equivalent by each Country of Origin and that the data contributor can provide their copies upon request, and

1.2 All co-authors on the data’s primary publication have consented to share the data in the GeOMe database.

ARTICLE 2 – USAGE OF DOWNLOADED GENETIC DATA AND ASSOCIATED METADATA

2.1 The Recipient shall be entitled to the Use of the Genetic Data and Associated Metadata for the Public Domain.

2.2 Should the Recipient intend to utilize the Genetic Data and Associated Metadata for Commercial Purposes they will seek consent of the Country of Origin, subject to the mutually agreed terms between the DIPnet contributor and Country of Origin.

2.3 When used for Commercial Purposes or in the Public Domain, data should be recognized as follows: “Data were made available through GeOMe (https://geome-db.org/)”.

2.4 Every attempt should be made to cite the original studies having contributed data to the new analyses. Attribution would include any publications first describing these data as well as reference to electronic depositions (such as DOI associated with dataDryad). This information may often be found in the “associatedReferences” field of the database.

2.5 The Recipient may transfer downloaded Genetic Data and Associated Metadata to Subsequent Recipients provided that Subsequent Recipients agree in writing to use the data under the terms of this agreement.

2.6 GeOMe does not undertake to monitor the rights of any Country of Origin for their Genetic Resources downstream from our database portal.
