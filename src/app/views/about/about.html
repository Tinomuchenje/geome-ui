About the Genomic Observatories Metadatabase

The Genomic Observatories Meta-Database (GeOMe) is a web-based database which captures metadata on biological samples, used for biodiversity inventories, population studies, and environmental metagenomics. GeOMe satisfies the requirements of the genomic observatories model, including capturing the who, what, where, and when associated with all samples. GeOMe draws on both the Darwin Core and the Minimum Information about any type of Sequence (MIxS) standards.

Getting Started

Welcome to GeOMe! For read access to public projects, you do not need an account and may create project-specific templates and query data. If you would like to contribute data to an existing project, you will need to contact the project administrator for access (see below). The project administrator of existing projects will create an account for you so that you will be able to load data. Creating a new project currently requires sending an email request to geome.help@gmail.com with a brief statement of what your project involves. The GeOMe team will be in touch about getting you setup.

Query GeOMe using "R"

You can query datasets that have been loaded to the GeOMe database using our R package. Please note, the R package was released for version 1 of the GeOMe site and has not yet been fully tested with the current release. If you encounter issues, please contact the development team.

The Biocode LIMS Plugin

Data uploaded to GeOMe can be manipulated in a laboratory information management system (LIMS) using a specially built LIMS plugin that operates within the Geneious environment. The purpose of the LIMS tool is to help manage lab and sequence analysis workflows. Note the name of the "LIMS plugin" for GeOMe is referred to as the "Biocode LIMS Plugin" since it was originally developed for the Moorea Bioocode Project. It has since been adapted for use in the GeOMe environment.
