About the Genomic Observatories Metadatabase

The Genomic Observatories Meta-Database (GeOMe) is a web-based database which captures metadata on biological samples, used for biodiversity inventories, population studies, and environmental metagenomics. GeOMe assigns persistent identifiers for all samples and sampling events and specifies the set of metadata attributes which satisfy the requirements of the genomic observatories model, including capturing the who, what, where, and when associated with all samples.

Template generation: what you need to do before you upload data

  • Generate a template (see Workbench -> "Generate Template") to create a spreadsheet template for your metadata
    • This option allows the user to create an excel file to store the metadata associated with each of their samples.

Users can upload several types of data

To upload data users much have an account. If you don't have one please send an email with you name, title and affiliation to geome.database@gmail.com

The following types of data are recognized in the GeOMe system:
  1. Collecting Event metadata
  2. Sample metadata
  3. Tissue metadata
  4. Photos of collecting event metadata
  5. Photos of sample metadata
  6. Published Sanger sequence data and the associated metadata (fasta format)
  7. Metadata associated with next-generation sequence data (fastq format)
    • In the process of uploading the metadata for your next-generation sequence data we create two files:
      1. the SRA metadata file
      2. the BioSample datasheet

This will make uploading your data to Genbank's Sequence Read Archive (SRA) database faster and easier .

* Note we are not currently accepting microsatellite datasets

Query GeOMe using "R"

You can query datasets that have been loaded to the GeOMe database using our R package. Please note, the R package was released for version 1 of the GeOMe site and has not yet been fully tested with the current release. If you encounter issues, please contact the development team.

Why FASTQ submissions for next generation sequence data?

GeOMe accepts FASTQ submissions because we want to archive unmanipulated sequence data that are free of filtering biases. By doing this we are reducing ascertainment bias (a site that is not a SNP in your dataset may be a SNP when combined with data from a different population). We are also avoiding the subjective choices that go into calling SNPs, such as thresholds for trimming, filtering, coverage and likelihood. Our objective is for future users of your data to be able to make these choices in the context of their own question and dataset.

The Biocode LIMS Plugin

Data uploaded to GeOMe can be manipulated in a laboratory information management system (LIMS) using a specially built LIMS plugin that operates within the Geneious environment. The purpose of the LIMS tool is to help manage lab and sequence analysis workflows. Note the name of the "LIMS plugin" for GeOMe is referred to as the "Biocode LIMS Plugin" since it was originally developed for the Moorea Bioocode Project. It has since been adapted for use in the GeOMe environment.
